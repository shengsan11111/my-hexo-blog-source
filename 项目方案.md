## **一、 项目背景与核心目标**

**1.1 原始需求**
用户需要一个公网可访问的网站，并希望开发一个本地工具，该工具能扫描指定文件夹内的所有Markdown文件，自动上传到网站并渲染展示，最终形成一个可供他人访问的个人知识库。

**1.2 澄清后的核心目标**
构建一个**自动化的工作流**，将分散在本地各处的Markdown笔记，通过简单的操作，发布为一个具备**整洁卡片式归档布局**的静态网站，并部署在用户完全掌控的公网服务器上。

## **二、 技术选型与架构决策**

经过需求澄清阶段的深入探讨，确定了以下关键技术决策：

| 决策项 | 选项 | 决策理由 |
| :--- | :--- | :--- |
| **网站形态** | 静态网站 | 部署简单、访问速度快、安全性高、资源消耗低。 |
| **静态网站生成器** | Hexo (基于Node.js) | 拥有丰富的主题生态，完美匹配“卡片式归档”的视觉需求，且与用户有基础的JavaScript技能栈匹配。 |
| **视觉主题偏好** | 卡片式、开箱即用 | 选择具有成熟卡片式主题的Hexo生态，实现快速启动与美观呈现。 |
| **服务器部署模式** | 基础架构式布置 | 从纯净的云服务器开始，以获得完全控制权，满足学习与定制需求。 |
| **内容同步模式** | 模式A (本地生成+上传成品) | 主流的Jamstack模式。在本地生成完整的静态网站文件，再上传到服务器，降低服务器负载，提升发布效率。 |
| **部署与上传方案** | 方案2: 直接文件上传 (SFTP/SCP) | 路径最短，反馈最快。通过编写Node.js脚本实现自动化，概念直观，易于理解和调试，适合项目初期快速获得“上线”正反馈。 |

## **三、 最终实现方案概述**

### **3.1 系统架构图**
```
[本地环境]
Markdown文件 ──(放入)──> Hexo项目 (`source/_posts/`)
                          │
                          ├─(执行 `node deploy.js`)──┐
                          │                          │
                          └─(生成)──> 静态网站 (`public/`) ──(通过SCP/SFTP上传)──┐
                                                                                │
[公网环境]                                                                      │
云服务器 (Nginx) <──(网站文件位于 `/usr/local/nginx/html/`) <────────────────┘
       │
       └─(用户访问)─> 浏览器 (通过IP: http://x.x.x.x)
```

### **3.2 核心组件说明**
1.  **内容生产端 (Hexo)**：
    *   **位置**：本地目录 `my-static-site`。
    *   **作用**：作为内容管理和静态网站生成引擎。负责将Markdown文件渲染为带主题的HTML页面。
    *   **主题**：采用了具备“卡片式归档”效果的Hexo主题（如 `hexo-theme-fluid`）。

2.  **自动化部署脚本 (`deploy.js`)**：
    *   **技术栈**：Node.js，使用 `node-ssh` 库。
    *   **工作流程**：
        1.  **生成**：在项目目录下执行 `hexo clean && hexo generate`，在本地生成最新的 `public` 文件夹。
        2.  **连接**：通过SSH连接到指定的公网服务器。
        3.  **清理**：清空服务器端网站目录的旧文件。
        4.  **上传**：将本地 `public` 文件夹内的所有文件递归上传至服务器。
        5.  **完成**：输出成功日志，提示访问地址。

3.  **公网服务器端**：
    *   **提供商**：腾讯云轻量应用服务器。
    *   **系统**：OpenCloudOS。
    *   **Web服务**：Nginx，网站根目录配置为 `/usr/local/nginx/html`。
    *   **访问方式**：通过服务器公网IP直接访问。

## 补充
- 后缀表达式本质上是二叉树的后序遍历

## 题目描述

给你一个字符串数组 `tokens`，表示一个根据逆波兰表示法表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

**注意：**

- 有效的算符为 `+`、`-`、`*` 和 `/`。
- 每个操作数（运算对象）都可以是一个整数或者另一个表达式。
- 两个整数之间的除法总是**向零截断**。
- 表达式中不含除零运算。
- 输入是一个根据逆波兰表示法表示的算术表达式。
- 答案及所有中间计算结果可以用 32 位整数表示。

## 示例

**示例 1：**

```
输入：tokens = ["2","1","+","3","*"]
输出：9
解释：该算式转化为常见的中缀算术表达式为：((2 + 1) * 3) = 9
```

**示例 2：**

```
输入：tokens = ["4","13","5","/","+"]
输出：6
解释：该算式转化为常见的中缀算术表达式为：(4 + (13 / 5)) = 6
```

**示例 3：**

```
输入：tokens = ["10","6","9","3","+","-11","*","/","*","17","+","5","+"]
输出：22
解释：该算式转化为常见的中缀算术表达式为：
  ((10 * (6 / ((9 + 3) * -11))) + 17) + 5
= ((10 * (6 / (12 * -11))) + 17) + 5
= ((10 * (6 / -132)) + 17) + 5
= ((10 * 0) + 17) + 5
= (0 + 17) + 5
= 17 + 5
= 22
```

## 约束条件

- `1 <= tokens.length <= 10^4`
- `tokens[i]` 是一个算符（`+`、`-`、`*` 或 `/`），或是在范围 `[-200, 200]` 内的一个整数

## 解题思路
### 采用栈解决逆波兰表达式/后缀表达式求值问题
### 为什么采用栈？
- 因为逆波兰表达式是后缀表达式，操作符在操作数后面，所以需要使用栈来存储操作数，直到遇到运算符时，才出栈两个操作数进行运算，相当于遇到运算符时，将三个元素进行运算，将运算结果压入栈中（类似于相邻项消除问题）

### 数学模拟过程（后缀转中缀逐步转化计算）
- 第一次：`10`、`6`、`9`、`3`
    - 遇到`+`：`9+3=12`
    - `+`栈结果：`10、6、(9+3)`
- 第二次：`10`、`6`、`(9+3)`、`-11` 
    - 遇到`*`：`((9+3)*-11)`
    - `*`栈结果：`10、6、((9+3)*-11)`
    - 遇到`/`：`6/((9+3)*-11)`
    - `/`栈结果：`10、6/((9+3)*-11)`
    - 遇到`*`：`10*(6/((9+3)*-11)`
    - `*`栈结果：`10*(6/((9+3)*-11)`
- 第三次：`10*(6/((9+3)*-11))`、`17`
    - 遇到`+`：`((10*(6/((9+3)*-11)))+17)`
    - `+`栈结果：`((10*(6/((9+3)*-11)))+17)`
- 第四次：`((10*(6/((9+3)*-11)))+17`、`5`
    - 遇到`+`：`((10*(6/((9+3)*-11)))+17)+5)`
    - `+`栈结果：`((10*(6/((9+3)*-11)))+17)+5)`
- 最后：`((10*(6/((9+3)*-11)))+17)+5=22`

### 代码模拟过程
- 定义一个栈存储整数（包含运算符计算结果和数字字符串），遍历tokens数组，依次判断是否为运算符或数字字符串，如果是运算符，需要出栈两个操作数，进行运算，将运算结果压入栈中；如果是数字字符串，直接压入栈中。

##  步骤

1. 定义一个栈，存储tokens当中的（纯数字字符串和带`-`的非纯数字字符串）转化成数字之后的结果和运算结果
2. 遍历tokens，不断取出每一个字符串
    - 首先利用strlen计算字符串长度len，定义一个整数temp_result，用于存储当前字符串的运算结果值/数字字符串转化成的结果值（不管当前字符串是算术运算符还是数字字符串必定都存在压栈操作）
        - len = 1：
            - 这个字符串必定是算数运算符、`0-9`其中一个数字
            - if语句判断是否为算术运算符
                - 如果是，则出栈两个元素
                    - 第一个出栈的元素作为运算符的第二个操作数
                    - 第二个出栈的元素作为运算符的第一个操作数
                    - 进行算术运算之后，将运算结果压入栈中
                - 如果不是，则将字符串转换成数字压入栈中 
        - len > 1：
            - 这个字符串必定是`-200` -- `200`
            - 直接将字符串转换成数字压入栈中
3. 栈中最后存放着整个表达式的结果值，将其弹出栈，返回即可

## 代码实现

```c
typedef struct
{
    int *nums;
    int length;
} mystack;
void pushStackTop(mystack *Mystack, int num)
{
    Mystack->nums[Mystack->length] = num;
    Mystack->length++;
}
void popStackTop(mystack *Mystack, int *num)
{
    *num = Mystack->nums[Mystack->length - 1];
    Mystack->length--;
}
int getStackTop(mystack *Mystack)
{
    return Mystack->nums[Mystack->length - 1];
}

int evalRPN(char **tokens, int tokensSize)
{
    mystack *Mystack = (mystack *)malloc(sizeof(mystack));
    Mystack->nums = malloc(tokensSize * sizeof(int));
    Mystack->length = 0;
    int len = tokensSize;
    for (int i = 0; i < len; i++)
    {
        int temp_len = strlen(tokens[i]);
        int temp_result;
        if (temp_len == 1)
        {
            if(strcmp(tokens[i],"+")==0)
            {
                int temp1, temp2;
                popStackTop(Mystack, &temp2);
                popStackTop(Mystack, &temp1);
                temp_result = temp1 + temp2;
            }
            else if(strcmp(tokens[i],"-")==0)
            {
                int temp1, temp2;
                popStackTop(Mystack, &temp2);
                popStackTop(Mystack, &temp1);
                temp_result = temp1 - temp2;
            }
            else if(strcmp(tokens[i],"*")==0)
            {
                int temp1, temp2;
                popStackTop(Mystack, &temp2);
                popStackTop(Mystack, &temp1);
                temp_result = temp1 * temp2;
            }
            else if(strcmp(tokens[i],"/")==0)
            {
                int temp1, temp2;
                popStackTop(Mystack, &temp2);
                popStackTop(Mystack, &temp1);
                temp_result = temp1 / temp2;
            }
            else
            {
                temp_result = atoi(tokens[i]);
            }
        }
        else
        {
            temp_result = atoi(tokens[i]);
        }
        pushStackTop(Mystack, temp_result);
    }
    return getStackTop(Mystack);
}
```

### 时间复杂度

`O(n)` 遍历tokens数组一次，每个字符串最多入栈一次出栈一次，所以时间复杂度为 `O(n)`。

### 空间复杂度

`O(n)` 栈最多存放 n 个整数（全部为数字时），所以空间复杂度为 `O(n)`。

## AI提出的优化角度
### 空间复杂度优化
1. 可以考虑使用数组代替栈，减少空间复杂度。
2. 不使用动态分配内存，直接在栈中分配内存，减少内存分配次数，提高效率（内存管理问题）
    - 需要手动管理栈的内存，避免内存泄漏。
    - 可以在程序结束时，free函数释放栈的内存。
### 性能优化
1. 直接字符检查，避免多次使用strcmp、strlen函数，提高效率
    - `char* token = tokens[i];`
    - `char first = token[0];`
    - `if (first == '+' || first == '-' || first == '*' || first == '/') && token[1] == '\0'`
### 代码量优化
1. 使用switch语句代替if-else语句，减少代码量

## 题目链接

[两数之和](https://leetcode.cn/problems/two-sum/)

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值** `target` 的那**两个**整数，并返回它们的**数组下标**。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。你可以按任意顺序返回答案。

### 示例

**示例 1：**
- **输入：** `nums = [2,7,11,15]`, `target = 9`
- **输出：** `[0,1]`
- **解释：** 因为 `nums[0] + nums[1] == 9`，返回 `[0, 1]`

**示例 2：**
- **输入：** `nums = [3,2,4]`, `target = 6`
- **输出：** `[1,2]`

**示例 3：**
- **输入：** `nums = [3,3]`, `target = 6`
- **输出：** `[0,1]`

### 约束条件

- `2 <= nums.length <= 10⁴`
- `-10⁹ <= nums[i] <= 10⁹`
- `-10⁹ <= target <= 10⁹`
- **只会存在一个有效答案**

---


## 初始解题思路

### 一开始采用了双重循环的方式来解决
首先外层循环遍历数组，找到第一个元素，再进入第二重循环下一个索引开始再次遍历后续数组元素，判断当前元素与第一个元素的和是否等于目标值，若等于，则返回两个元素的下标。若不等于，则继续遍历。

---

## 步骤

1. 外层遍历数组nums，存储第一个元素的下标值到return_array[0]
2. 第二层从return_array[0]+1开始遍历数组，对于每个元素nums[i]，判断当前元素与第一个元素的和是否等于目标值，若等于，则返回两个元素的下标。若不等于，则继续遍历。

---

## 代码实现

### 双重循环
```c
int *twoSum(int *nums, int numsSize, int target, int *returnSize)
{
    int i = 0, j = 0;
    int *return_array = malloc(2 * sizeof(int));
    for(i=0;i<numsSize;i++){
        return_array[0]=i;
        for(j=i+1;j<numsSize;j++){
            if(nums[i]+nums[j]==target){
                return_array[1]=j;
            }
        }
        if(return_array[0]<return_array[1]){
            break;
        }
    }
    *returnSize = 2;
    return return_array;
}
```
### 时间复杂度

O(n²) 最坏的情况下是每个元素都需要遍历其他所有元素，时间复杂度为 O(n²)

### 空间复杂度

O(1) 只需要存储两个元素的下标值，空间复杂度为 O(1)


## 最新优化解题思路

### 采用哈希表 + 线性探查的方式来解决
创造一个哈希表当中存放结构体HashNode，结构体当中需要包含元素的值key、元素的下标index、是否被使用过used。遍历数组Nums，对当前元素的补数（和target差值）通过线性探查法查找判断哈希表当中是否存在补数，若存在，则返回补数对应的下标（也就是补数对应的结构体的index）和当前下标 i ，若不存在，则就需要以线性探查法的方式将当前元素及其下标 i 存入哈希表，但需要注意：哈希表当中的key唯一，不可以重复插入相同key值。

---

## 步骤

1. 构造一个结构体HashNode，用于存储哈希表中的每个元素
   - key：元素的值
   - index：元素的下标
   - used：是否被使用过，若为1，则说明该元素已被使用过，若为0，则说明该元素未被使用过
2. 构造一个哈希函数hashCheck，用于计算元素的哈希值，哈希函数的实现为：(key % size + size) % size，用于找到起始探查位置
3. 构造一个哈希表，大小为HASH_SIZE，用于存储数组中的元素，需要初始化全部为0
4. 遍历数组nums，对于每个元素nums[i]
   - 先计算当前元素的补数poor = target - nums[i]
   - 在哈希表中查找补数poor是否存在于哈希表中（从poor的哈希起始位置开始线性探查）
   - 若存在，返回补数对应的下标（也就是补数对应的结构体的index）和当前下标 i 
   - 若不存在，就需要将当前元素nums[i]及其下标 i 存入哈希表
   - 同样采用线性探查法，将当前元素nums[i]及其下标 i 存入哈希表（从nums[i]的哈希起始位置开始线性探查）
5. 返回数组result_array



---


### 哈希表 + 线性探查

```c
#define HASH_SIZE 20011

typedef struct {
    int key;
    int index;
    int used;
} HashNode;

int hashCheck(int key, int size) {
    return (key % size + size) % size;
}

int* twoSum(int* nums, int numsSize, int target, int* returnSize) {
    HashNode* hashArray = (HashNode*)calloc(HASH_SIZE, sizeof(HashNode));
    int* result_array = (int*)malloc(2 * sizeof(int));
    *returnSize = 2;

    for (int i = 0; i < numsSize; i++) {
        int poor = target - nums[i];
        int h = hashCheck(poor, HASH_SIZE);
        while (hashArray[h].used) {
            if (hashArray[h].key == poor) {
                result_array[0] = hashArray[h].index;
                result_array[1] = i;
                free(hashArray);
                return result_array;
            }
            h = (h + 1) % HASH_SIZE;
        }

        h = hashCheck(nums[i], HASH_SIZE);
        while (hashArray[h].used) {
            if (hashArray[h].key == nums[i]) {
                break;
            }
            h = (h + 1) % HASH_SIZE;
        }
        hashArray[h].key = nums[i];
        hashArray[h].index = i;
        hashArray[h].used = 1;
    }

    free(hashArray);
    return result_array;
}
```

---

### 时间复杂度

O(n) 遍历数组一次，哈希表查找和插入均为 O(n)

### 空间复杂度

O(n) 最坏情况下需要存储 n 个元素

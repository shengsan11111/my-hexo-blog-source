# 项目简介

## 一、项目概览

### 项目类型
**前端静态站点项目** —— 基于 Hexo 框架构建的个人知识库/博客系统，具备一键自动化部署能力。

### 核心定位
将本地分散的 Markdown 笔记文件，通过简单的操作自动发布为具备**卡片式归档布局**的静态网站，部署至用户完全掌控的公网服务器，形成可公开访问的个人知识库。

### 主要技术栈

| 类别 | 技术 | 版本 | 说明 |
|:---|:---|:---|:---|
| **静态站点生成器** | Hexo | ^8.0.0 | 基于 Node.js 的快速、简洁且高效的博客框架 |
| **主题引擎** | hexo-theme-fluid | ^1.9.9 | 卡片式归档主题，开箱即用 |
| **Markdown 渲染** | hexo-renderer-marked | ^7.0.0 | Markdown 转 HTML |
| **模板引擎** | hexo-renderer-ejs | ^2.0.0 | EJS 模板渲染 |
| **样式预处理** | hexo-renderer-stylus | ^3.0.1 | Stylus CSS 预处理 |
| **SSH 部署** | node-ssh | ^13.2.1 | Node.js SSH/SFTP 客户端 |
| **辅助传输** | ssh2 / scp2 | ^1.17.0 / ^0.5.0 | 底层 SSH 通信支持 |
| **运行环境** | Node.js | - | 项目运行基础 |

## 二、功能分析

### 2.1 主要功能

1. **Markdown 内容管理**：将 Markdown 文件放入 `source/_posts` 目录，Hexo 自动渲染为带主题样式的 HTML 页面
2. **卡片式首页展示**：文章以卡片形式排列，支持按日期排序，视觉整洁
3. **完整的博客体系**：内置归档页、分类页、标签页、关于页等标准页面
4. **一键自动化部署**：执行单条命令即可完成「生成 → 清理 → 上传」全流程
5. **代码高亮**：集成 highlight.js，支持代码块语法高亮与行号显示
6. **依赖自动更新**：配置 GitHub Dependabot，每日检查 npm 依赖更新

### 2.2 核心模块

```
my-static-site/
├── source/_posts/          # 内容层：存放 Markdown 文章
├── scaffolds/              # 模板层：文章/页面/草稿模板
│   ├── post.md             #   文章模板（含 Front Matter）
│   ├── page.md             #   页面模板
│   └── draft.md            #   草稿模板
├── _config.yml             # 全局配置：站点信息、URL规则、部署设置
├── _config.fluid.yml       # 主题配置：导航栏、卡片样式、页面菜单
├── deploy.js               # 部署引擎：自动化构建+上传脚本
├── themes/                 # 主题目录
└── .github/dependabot.yml  # CI/CD：依赖自动化更新
```

### 2.3 功能协作流程

```
写作阶段:
  Markdown 文件 → 放入 source/_posts/

发布阶段 (执行 node deploy.js):
  ┌─────────────┐    ┌──────────────┐    ┌─────────────┐
  │ hexo clean  │ → │ hexo generate│ → │ SSH 连接服务器 │
  │ 清理旧产物   │    │ 生成静态页面  │    │ 清空远程目录  │
  └─────────────┘    └──────────────┘    │ 上传 public/ │
                                         └─────────────┘

访问阶段:
  浏览器 → Nginx (公网服务器) → HTML 静态页面
```

## 三、问题解决

### 3.1 解决的核心痛点

- **笔记分散难管理**：本地各处的 Markdown 文件无法统一展示和分享
- **建站门槛高**：传统建站需要掌握服务器运维、数据库、后端开发等复杂技能
- **发布流程繁琐**：每次更新内容需要手动生成、手动上传多个步骤
- **缺乏美观展示**：原始 Markdown 缺乏排版美感和导航结构

### 3.2 方案优势与创新点

| 对比维度 | 传统方案 | 本项目方案 |
|:---|:---|:---|
| **部署方式** | 手动 FTP/Git 推送 | 一条命令全自动（生成+清理+上传） |
| **技术复杂度** | 需要后端+数据库+运维 | 纯静态，零数据库，零服务端逻辑 |
| **服务器负载** | 动态渲染消耗资源 | Nginx 直出静态文件，极低资源占用 |
| **访问速度** | 取决于服务器性能 | 静态文件秒开，CDN 友好 |
| **安全性** | 存在 SQL 注入等风险 | 无动态接口，攻击面极小 |
| **内容迁移** | 依赖数据库导出 | Markdown 文件即内容，随时可迁 |

### 3.3 目标用户群体

- 希望搭建个人知识库/技术博客的开发者
- 有大量 Markdown 笔记需要统一管理和展示的用户
- 希望完全掌控服务器、不依赖第三方平台（如 GitHub Pages）的用户
- 追求简洁高效工作流的技术爱好者

## 四、快速上手

### 4.1 环境要求

- **Node.js** >= 14（推荐 LTS 版本）
- **npm** 或 **yarn** 包管理器
- 一台已配置好 **Nginx** 的公网云服务器（用于部署）

### 4.2 安装步骤

```bash
# 1. 进入项目目录
cd my-static-site

# 2. 安装依赖
npm install
# 或
yarn install

# 3. 启动本地预览服务器
npm run server
# 或
hexo server

# 浏览器访问 http://localhost:4000 即可预览
```

### 4.3 关键配置说明

#### 站点配置 — [`_config.yml`](hexo-blog/my-static-site/_config.yml)

```yaml
# 站点基本信息
title: Hexo              # 网站标题
subtitle: ''             # 副标题
description: ''          # 网站描述
author: John Doe         # 作者
language: en             # 语言
url: http://example.com  # 网站URL

# 主题选择
theme: fluid            # 使用 Fluid 卡片式主题

# 文章永久链接格式
permalink: :year/:month/:day/:title/
```

#### 主题配置 — [`_config.fluid.yml`](hexo-blog/my-static-site/_config.fluid.yml)

```yaml
navbar:
  blog_title: 我的知识库   # 网站名称

index:
  post_style: card        # 卡片式文章列表
  post_order_by: date     # 按日期排序
```

#### 部署配置 — [`deploy.js`](hexo-blog/my-static-site/deploy.js)

```javascript
const config = {
    host: '81.71.163.202',     // 服务器公网IP
    username: 'root',           // 登录用户名
    password: '********',       // 登录密码（建议改用SSH密钥）
};
const remoteDir = '/usr/local/nginx/html';  // 服务器网站根目录
```

> ⚠️ **安全提醒**：生产环境中建议将密码替换为 SSH 密钥认证，避免在代码中硬编码敏感信息。

### 4.4 最简使用示例

**发布一篇文章只需两步：**

```bash
# 第一步：将 Markdown 文件放入文章目录
#    my-static-site/source/_posts/我的新文章.md

# 第二步：执行部署命令
node deploy.js

# 输出示例：
# 🚀 开始部署...
# ✅ Hexo 生成完成
# 🔗 连接服务器...
# 🧹 清理远程目录...
# 📤 上传文件中...
# ✅ 上传完成: ./public/index.html
# ...
# 🎉 部署成功！
# 👉 访问地址: http://您的服务器IP
```

### 4.5 可用命令速查

| 命令 | 说明 |
|:---|:---|
| `npm run server` 或 `hexo s` | 启动本地预览服务器（端口 4000） |
| `npm run build` 或 `hexo g` | 生成静态文件到 `public/` 目录 |
| `npm run clean` 或 `hexo clean` | 清除缓存和生成的文件 |
| `node deploy.js` | 一键构建并部署到远程服务器 |
| `hexo new "文章标题"` | 创建新文章草稿 |
